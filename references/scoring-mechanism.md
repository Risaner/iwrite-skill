# 机器批改系统评分机制深度解析

## iWrite 评分原理（北外梁茂成教授设计）

### 核心方法
- **链语法（Link Grammar）+ 有监督机器学习（SML）** 的混合方法
- 从大规模英语本族语者语料库和学习者错误语料库获取数据

### 错误分级（按严重程度）
1. 句法错误（最严重）
2. 词法错误
3. 搭配错误
4. 技术规范错误（最轻）

### 内容评价（国内首创）
- 基于**母语语料库**和**英语联想词库**
- 测评**切题度（global coherence）** 和 **连贯性（local coherence）**
- 相邻句子/段落间的连贯性与人工评判高度相关

## 批改网（pigai.org）评分原理

### 核心方法
- 将学习者语料与标准语料库（母语用户语言资源库）比对
- 测量两者之间的"距离"数据
- **192 个语言分析维度** 映射为分数

### 五大模块
1. **作文分析模块**：分析词汇、词块、搭配、句型
2. **对比模块**：与知识运维库对比，找亮点和不足
3. **打分模块**：按教师设置的权重打分
4. **数据挖掘模块**：生成图文报表
5. **知识运维库**：提供对比数据支持

### 中式英语识别
- 大数据分析 + 深度学习
- 建立可实时更新的错误库
- 能识别文化差异造成的语言用法问题

## 学术论文实证数据

### iWrite 平台实验（2026年宿迁学院研究）
- 124 名学生，12 周准实验
- 实验组（iWrite + 教师指导）vs 控制组（传统模式）
- **结果**：实验组后测成绩更高，词汇产出量、学术词汇占比、句法复杂度均优化
- **个案**：学生 V1→V7 迭代，78.5 分 → 88.2 分

### 批改网实证
- 150 次修改：30 分 → 73.5 分
- 作文长度是最显著的相关变量（流利度指标）
- 句子结构复杂度、关系代词使用数量是关键特征

## 系统局限性（利用点）

1. **内容理解有限**：系统主要靠关键词匹配判断切题度，不真正理解语义
2. **结构词权重高**：大量连接词 = 结构好，即使逻辑上不需要
3. **字数敏感**：字数是回归模型中的强特征
4. **格式敏感**：标点空格、段落格式直接影响技术规范分
5. **词汇偏好可利用**：系统对母语语料库中高频出现的"高级"表达有加分
